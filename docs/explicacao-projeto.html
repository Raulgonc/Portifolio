Visão Geral do Projeto

O projeto é um portfólio profissional de página única (SPA) construído com Angular 19. A estética foi inspirada em Cowboy Bebop — a série de animê japonesa de 1998 — que mistura jazz noir, viagens espaciais e um visual retro-futurista único.

Toda a interface usa uma paleta de espaço profundo (fundos quase pretos), cores quentes em âmbar/ferrugem e tipografia monospace para transmitir a sensação de um terminal de bordo de uma nave espacial dos anos 2070.

Tecnologia Principal

Angular 19 com Standalone Components — arquitetura moderna sem NgModules, cada componente é independente e importa só o que precisa.

Estilo

SCSS com variáveis CSS globais (:root) definindo toda a paleta e tipografia — fácil de alterar em um único lugar.

Roteamento

Angular Router com lazy loading — a página Home só carrega quando o usuário acessa a rota, melhorando a performance inicial.

Fontes (Google Fonts)

Orbitron para títulos, Space Mono para texto e código, Bebas Neue disponível para destaques decorativos.

Angular Signals: O projeto usa a nova API de Signals do Angular 19 (ex: signal(false) na Navbar para o estado de scroll) — mais performático que o tradicional EventEmitter para estados locais simples.

Componente: Navbar

A barra de navegação é fixa no topo (não rola com a página) e muda de aparência conforme o usuário desce a tela.

Comportamento de Scroll

Usa o decorator @HostListener('window:scroll') para escutar o evento de scroll do navegador. Quando o scroll passa de 50px, o Signal scrolled muda de false para true, adicionando a classe .scrolled ao elemento <nav>. 

scrolled = signal(false);

// Escuta o scroll da janela inteira
@HostListener('window:scroll')
onScroll() {
  this.scrolled.set(window.scrollY > 50);
}

Estado Inicial (topo)

Fundo transparente, padding maior, sem borda inferior — se mistura ao Hero sem quebrar o visual espacial.

Estado Após Scroll (.scrolled)

Fundo escuro com backdrop-filter: blur(10px), padding menor, borda inferior âmbar — fica legível sobre qualquer seção.

Componente: Hero (Starfield)

A seção Hero ocupa 100% da altura da tela e é o primeiro impacto visual. O elemento mais técnico aqui é o starfield animado desenhado diretamente com a Canvas API do navegador.

Como Funciona o Starfield

O Angular cria uma referência ao elemento <canvas> do HTML via @ViewChild. No ngOnInit, o componente inicializa o contexto 2D e começa o loop de animação.

ngOnInit
→
Cria 180 estrelas
(posição/tamanho/speed aleatórios)
→
requestAnimationFrame loop
→
Move cada estrela para baixo
Volta ao topo ao sair da tela
// Cada estrela tem 5 propriedades:
{
  x:       // posição horizontal
  y:       // posição vertical
  r:       // raio (0.2 a 1.7px — tamanhos variados)
  speed:   // velocidade de queda (0.05 a 0.35)
  opacity: // opacidade (0.3 a 1.0)
}
Importante: O ngOnDestroy cancela o requestAnimationFrame com cancelAnimationFrame(this.animId). Isso evita memory leak — se o componente for destruído, o loop de animação para.

Componente: About

Seção de apresentação pessoal com layout em grid de duas colunas: texto à esquerda e um cartão visual à direita simulando um arquivo JSON aberto num terminal.

Coluna Esquerda — Texto

Parágrafo de apresentação + detalhes em formato de tabela com labels em âmbar (STATUS, LOCATION, FOCUS). O status disponível tem um ponto verde pulsante via CSS.

Coluna Direita — Terminal Card

Imita a interface de um editor de código com cabeçalho (dots vermelho/amarelo/verde) e syntax highlighting aplicado com classes CSS: .key, .str, .num, .bool.

// Exemplo do card terminal em HTML
<pre class="card-code"><code>
  { "name": "Seu Nome", ... }
</code></pre>
Responsividade: Em telas menores que 768px o grid passa de 2 colunas para 1 coluna via media query — as seções ficam empilhadas verticalmente.

Componente: Skills

Lista as habilidades técnicas organizadas em três categorias (Frontend, Backend, Tools) com barras de progresso estilizadas.

Estrutura de Dados

As skills são um array de objetos com interface TypeScript, definidos diretamente no componente: 

interface Skill {
  name:     string;   // "Angular"
  level:    number;   // 85 (representa 85%)
  category: string;   // "Frontend" | "Backend" | "Tools"
}

Renderização com @for

Usa a nova sintaxe de controle de fluxo do Angular 17+ (@for) — mais legível que *ngFor e sem precisar importar CommonModule: 

@for (skill of byCategory(cat); track skill.name) {
  <div class="skill-bar-fill"
       [style.width.%]="skill.level"></div>
}

O binding [style.width.%]="skill.level" aplica a largura diretamente como percentual via property binding do Angular — sem precisar de string interpolation.

Barra de Progresso

A barra tem três camadas visuais:

CamadaEstilo
Track (fundo)Fundo escuro semitransparente
Fill (preenchimento)Gradiente âmbar-dim → âmbar com box-shadow glow
Ponteiro (::after)4px brilhante no fim da barra com glow intenso

Usa @if para só renderizar links se eles existirem no objeto:

@if (project.link) {
  <a [href]="project.link" target="_blank">↗ Ver projeto</a>

Componente: Contact

Seção final com grid de duas colunas: links de contato à esquerda e formulário de mensagem à direita.

Links de Contato

Email, GitHub e LinkedIn com ícones simbólicos e efeito de translação horizontal no hover (translateX(6px)).

Formulário

Campos Nome, Email e Mensagem com estilo dark. O submit usa (submit)="$event.preventDefault()" — pronto para conectar a um serviço de email.

Botão com Clip-path

O botão de envio usa clip-path: polygon() para ter o canto superior direito e inferior esquerdo cortados diagonalmente — detalhe estético retro-futurista recorrente na UI:

clip-path: polygon(
  0 0,
  calc(100% - 10px) 0,  /* corte superior direito */
  100% 10px,
  100% 100%,
  10px 100%,            /* corte inferior esquerdo */
  0 calc(100% - 10px)
);

Página Home e Roteamento

A página Home é o "montador" — ela importa todos os componentes e os coloca em sequência. Está associada à rota raiz '/'.

Fluxo Completo de Carregamento

→
Lazy load

→

→
Página
renderizada

Lazy Loading

A rota usa loadComponent em vez de importar diretamente:

{
  path: '',
  loadComponent: () =>
    import('./pages/home/home').then(m => m.Home),
}

Isso significa que o bundle da página Home (e todos seus componentes) só é baixado quando o usuário acessa a rota — o bundle inicial fica menor e a aplicação carrega mais rápido.
